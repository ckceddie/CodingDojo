<h1>Forms</h1>
<p>
Forms are <strong>one of the most important</strong> HTML tags you will learn. They are responsible for <strong>all the data exchange</strong> between the user (front-end) and the server (back-end). So it's very important that you have a good understanding of how to build them.</p>
<p>
A form's job is to<strong> take in user input </strong>and <strong>send it to the back-end to be processed</strong>. A form is declared by using the <strong>&lt;form&gt; tag</strong>, which will have <strong>attributes action and method</strong> that decide <strong>where</strong> the form information gets sent, and <strong>how</strong> it gets sent respectively. (Don't worry about the values we're using for these attributes yet, this is covered during your first stack.)</p>
<p>Taking in the input is done by using<strong>&nbsp;input fields</strong>, usually designated by the&nbsp;<strong>&lt;input&gt;</strong>&nbsp;tag. Depending on the type of information required, the way in which we take in the information may be different. This is sometimes designated by a<strong>&nbsp;type attribute</strong>, and other times by a different tag. Each input will typically also have a label (<strong>&lt;label&gt;</strong>) that is the name of the field.&nbsp;To make sure that a specific label is linked/associated to a specific input element, we must add corresponding<strong><em>&nbsp;for attribute</em></strong>&nbsp;on the label with input's&nbsp;<strong><em>id attribute</em></strong>.&nbsp;Including a label tag around the input field is a convention that allows us to click on the label to focus on the input field.</p>
<p>A&nbsp;<strong>name attribute</strong>&nbsp;also will typically go hand in hand with your input tags. They are used mainly for sending form data to the back end so don't worry about the name attribute for now. It will be explained further in your first stack.</p>
